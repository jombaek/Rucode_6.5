## A. Младший бит

<table>
      <tr><td>Ограничение времени</td><td>1 секунда</td></tr>
      <tr><td>Ограничение памяти</td><td>512Mb</td></tr>
      <tr><td>Ввод</td><td>стандартный ввод или input.txt</td></tr>
      <tr><td>Вывод</td><td>стандартный вывод или output.txt</td></tr>
</table>

Заданы $n$ целых чисел. Определить, сколько чисел среди заданных имеют единицу в самом младшем бите в двоичной записи.

### Формат ввода

Первая строка входных данных содержит целое число $n$ — количество чисел $(1 \leq n \leq 1000)$.
Вторая строка содержит $n$ целых чисел, по абсолютной величине не превосходящих $10^5$ — заданные числа.

### Формат вывода

Выведите количество чисел среди заданных, у которых самый младший (самый правый) бит в двоичной записи равен 1.

### Пример

**Ввод**
```c++
5 14
```
**Вывод**
```c++
19
```

## B. Вещественные и целые

<table>
      <tr><td>Ограничение времени</td><td>1 секунда</td></tr>
      <tr><td>Ограничение памяти</td><td>512Mb</td></tr>
      <tr><td>Ввод</td><td>стандартный ввод или input.txt</td></tr>
      <tr><td>Вывод</td><td>стандартный вывод или output.txt</td></tr>
</table>

На вход подаётся вещественное число. Требуется его округлить до ближайшего целого, которое его не превосходит, до ближайшего целого, которое его превосходит, и в соответствии с правилами округления (в частности, 0.5 округляется в направлении от нуля), и вывести все три целых числа в указанном порядке.

### Формат ввода

Одно вещественное число в интервале от $-1000$ до $1000$ с непустой дробной частью.

### Формат вывода

Выведите три целых числа: первое — ближайшее целое, не превосходящее заданного вещественного числа, второе — ближайшее целое, превосходящее заданное вещественное число, третье — ближайшее целое по правилам округления.

### Примеры

**Ввод**
```c++
1.234500000
```
**Вывод**
```c++
1 2 1
```
**Ввод**
```c++
1.500000000
```
**Вывод**
```c++
1 2 2
```
**Ввод**
```c++
-3.500000000
```
**Вывод**
```c++
-4 -3 -4
```

## C. Подбор лучшего защитника

<table>
      <tr><td>Ограничение времени</td><td>1 секунда</td></tr>
      <tr><td>Ограничение памяти</td><td>512Mb</td></tr>
      <tr><td>Ввод</td><td>стандартный ввод или input.txt</td></tr>
      <tr><td>Вывод</td><td>стандартный вывод или output.txt</td></tr>
</table>

На вход подаётся вещественное число. Требуется его округлить до ближайшего целого, которое его не превосходит, до ближайшего целого, которое его превосходит, и в соответствии с правилами округления (в частности, 0.5 округляется в направлении от нуля), и вывести все три целых числа в указанном порядке.

### Формат ввода

В  первой строке записаны два целых числа $n$, $m$ $(1 \leq n, m \leq 2 \cdot 10^5)$ — количество кандидатов в основу клуба и количество запросов.
Во второй строке записаны $n$ целых чисел в невозрастающем порядке, каждое по абсолютной величине не превышает $10^9$ — коэффициенты защитной эффективности игроков.
В следующих $m$ строках даны запросы. Каждый запрос содержит одно целое число $q$, по абсолютной величине не превышающее $10^9$.

### Формат вывода

Для каждого запроса выведите минимальный номер футболиста с коэффициентом защитной эффективности, меньшим $q$. Если такого футболиста в распоряжении скаута нет, вместо номера выведите «NO SOLUTION» (без кавычек).

### Пример

**Ввод**
```c++
11 9
20 17 16 14 6 4 0 -3 -5 -14 -17
8
-5
1
6
19
16
4
-17
-19
```
**Вывод**
```c++
5
9
7
5
2
3
6
11
NO SOLUTION
```

## D. Германн и рулетка

<table>
      <tr><td>Ограничение времени</td><td>1 секунда</td></tr>
      <tr><td>Ограничение памяти</td><td>512Mb</td></tr>
      <tr><td>Ввод</td><td>стандартный ввод или input.txt</td></tr>
      <tr><td>Вывод</td><td>стандартный вывод или output.txt</td></tr>
</table>

Недавно в неизвестных ранее рукописях Пушкина, которые нашлись у одного из коллекционеров, обнаружили черновики исходного варианта «Пиковой дамы». В нём Германн играл на рулетке и интересовался выигрышным набором номеров. Всё, что он смог выведать у графини — что к волшебному выигрышу может привести возрастающая последовательность номеров длины K. На рулетке, на которой играет Германн, $N$ номеров от 1 до $N$.
Германн хочет изучить возможный список. Поэтому он просит вас по заданным числам $N$ и $K$ вывести все возрастающие последовательности длины $K$ из чисел от 1 до $N$ в лексикографическом порядке.

### Формат ввода

Вход содержит одну строку с двумя целыми числами $N$ и $K$. $(1 \leq K, N \leq 100)$. Гарантируется, что число выведенных последовательностей не превысит 5678.

### Формат вывода

Выведите требуемый список последовательностей в лексикографическом порядке. Последовательности выводятся по одной на строку.

### Примеры

**Ввод**
```c++
3 2
```
**Вывод**
```c++
1 2
1 3
2 3
```
**Ввод**
```c++
4 1
```
**Вывод**
```c++
1
2
3
4
```

## E. Старый ноутбук

<table>
      <tr><td>Ограничение времени</td><td>10 секунд</td></tr>
      <tr><td>Ограничение памяти</td><td>32Mb</td></tr>
      <tr><td>Ввод</td><td>стандартный ввод или input.txt</td></tr>
      <tr><td>Вывод</td><td>стандартный вывод или output.txt</td></tr>
</table>

Для целого числа $X$ Байтазар придумал новую функцию: $B(x)$ — количество простых чисел, меньших или равных числа $X$. Чтобы исследовать поведение $B(x)$, Байтазару нужна программа, которая по заданному $X$ вычисляет $B(X). Только вот ноутбук Байтазара совсем старый, и он может выделить всего 32 MiB памяти.
Напишите программу, которая нужна Байтазару.

### Формат ввода

Входной файл содержит одно целое число $X$ $(2 \leq X \leq 2 \cdot 10^8)$.

### Формат вывода

Выведите одно целое число — значение $B(x)$.

### Примеры

**Ввод**
```c++
2
```
**Вывод**
```c++
1
```
**Ввод**
```c++
20
```
**Вывод**
```c++
8
```

## F. Перетягивание каната

<table>
      <tr><td>Ограничение времени</td><td>1 секунда</td></tr>
      <tr><td>Ограничение памяти</td><td>512Mb</td></tr>
      <tr><td>Ввод</td><td>стандартный ввод или input.txt</td></tr>
      <tr><td>Вывод</td><td>стандартный вывод или output.txt</td></tr>
</table>

В чемпионате по перетягиванию каната участвует $N$ человек, $i$-й имеет силу $p_i$. Для показательных выступлений требуется разделить их на две команды так, чтобы суммарная сила участников из первой команды была равна суммарной силе участников из второй. Можно ли это сделать?

### Формат ввода

Первая строка входных данных содержит целое положительное число $N$, не превышающее 100 — количество участников чемпионата. Вторая строка содержит $N$ целых положительных чисел $p_i$, не превышающих 100 — силу каждого участника.

### Формат вывода

Выведите “YES”, если участников можно разбить на две команды с равной суммарной силой, и “NO” в противном случае.

### Примеры

**Ввод**
```c++
4
3 9 4 2
```
**Вывод**
```c++
YES
```
**Ввод**
```c++
1
42
```
**Вывод**
```c++
NO
```
**Ввод**
```c++
2
20 23
```
**Вывод**
```c++
NO
```

## G. Максимальная компонента

<table>

      <tr><td>Вывод</td><td>стандартный вывод или output.txt</td></tr>

Вам задан неориентированный граф с $N$ вершинами и $M$ рёбрами $(1 \leq N \leq 20 000, 1 \leq M \leq 200 000)$. В графе отсутствуют петли и кратные рёбра.
Найдите компоненту связности заданного графа с наибольшим количеством вершин и выведите соответствующее количество вершин.

### Формат ввода

Первая строка входа содержит числа $N$ и $M$. Каждая из следующих $M$ строк содержит описание ребра — два целых числа из диапазона от 1 до $N$ — номера концов ребра.

### Формат вывода

Выведите одно целое число — число вершин в максимальной компоненте связности графа.

### Пример

**Ввод**
```c++
4 2
1 2
3 4
```
**Вывод**
```c++
2
```
